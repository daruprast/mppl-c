4. Sebutkan dan jelaskan jenis version control system yang digunakan dalam pengembangan perangkat lunak.
   --> Dalam pengembangan perangkat lunak, ada yang namanya VCS atau version control system, VCS adalah sistem yang digunakan untuk mencatat perubahan terhadap kode/program dari waktu ke waktu, sehingga pengembang dapat mengelola revisi, berkolaborasi, dan  
	mengembalikan versi sebelumnya jika terjadi kesalahan. Ada beberapa jenis VCS yaitu :
	a. Local Version Control System (LVCS)
	   -> LVCS adalah VCS secara lokal yang menyimpan versi file secara lokal di komputer pengguna dengan menggunakan database sederhana untuk melacak perubahan file. Kelemahan dari LVCS ini adalah tidak mendukung kolaborasi tim karena semua data ada di satu  
		komputer. Contoh dari LVCS yaitu RCS atau Revision Control System

	b. Centralized Version Control System (CVCS)
	   -> CVCS adalah version control yang semua versi kodenya disimpan di satu server pusat sehingga pengembang bekerja dengan salinan lokal dan sinkronisasi (commit/update) ke server pusat. Kelemahan dari CVCS adalah Jika server pusat down atau rusak, semua orang 
		tidak bisa bekerja atau kehilangan riwayat. Contoh dari CVCS yaitu Subversion (SVN), Perforce, CVS (Concurrent Versions System)

	c. Distributed Version Control System (DVCS)
	   -> DVCS mirip seperti CVCS hanya saja bedanya adalah terdapat 2 versi kode/repositori yakni di Lokal dan Server sehingga setiap pengembang memiliki salinan lengkap dari seluruh riwayat proyek dan sinkronisasi dilakukan dengan push/pull ke repositori pusat 
		atau ke sesama pengembang. Contoh dari DVCS yakni Git, Mercurial, Bazaar.
